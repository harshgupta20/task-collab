# Security Policy

## Supported Versions
Security updates or patches will only be provided for the latest version of the project.  
Users are encouraged to always use the most recent commit or release.

## Reporting a Vulnerability
This project is fully open-source and community-driven.  
If you discover a security vulnerability:

1. **Do NOT expect private reporting.**  
   Issues and fixes are handled via Pull Requests (PRs) or GitHub Issues.

2. **Submit a Pull Request** with the fix:  
   - Provide a detailed description of the problem.  
   - Include steps to reproduce if applicable.  
   - Explain the security impact of the issue.

3. **Label the PR or Issue as `security`** for visibility.

## Handling of Reported Vulnerabilities
- Pull Requests with fixes will be reviewed by maintainers or community members.  
- Once verified, the fix will be merged and released.  
- Users should always update to the latest version to stay secure.

## Security Best Practices for Contributors
- Do not commit sensitive data (keys, tokens, passwords).  
- Follow secure coding principles.  
- Validate and sanitize inputs where appropriate.  
- Review third-party dependencies for safety.

## Disclaimer
This project is for educational and learning purposes.
